## 통합/이식 우선순위 평가

### 결론: **현재 상태 유지 권장 (즉시 통합 불필요)**

이유:
1. **US Lab은 자체 완비**. Backtest + Forward 2탭으로 시장-적합한 사용 흐름이 이미 정착.
2. **KR 9전략 모델은 KR 시장 특화** — 한국 시장 LowVol/Mom/Sector 등의 팩터로 설계됨. SPY 시장에 그대로 옮기면 알파 가설이 검증되지 않은 채 실거래로 직행할 위험.
3. **양 시장의 Lab 목적이 다름**:
   - KR: 9전략 동시 forward 운영 (paper) — 실험 비교 도구
   - US: 백테스트 → 단일 전략 forward — 검증 후 운영 도구
4. **공통 UI 컴포넌트화 가치 낮음**. 각자 도메인 특화 UX가 더 효율적.

### A. 백테스트 엔진 공통화 (선택적, 큰 작업)
- KR `strategy_lab.js` + KR `web/lab_simulator.py` ↔ US `web/backtest/*`
- 장점: 동일 백테스트 인프라, 결과 직접 비교 가능
- 단점: 두 시장의 데이터 모델이 달라 추상화 비용 높음

### B. KR 9전략 → US 적합도 검증 (별도 연구 작업)
- 9전략 중 시장 무관 전략 (mean_reversion, vol_regime 등) → SPY 데이터로 백테스트
- 결과 양호 시 US Lab Forward 탭에 추가 후보로 등록
- **별건 — 트레이딩 리서치 작업, UI 통합 아님**

### C. Lab 페이지 nav 통일 (소규모)
- 현재 양 시장 `/lab` 모두 nav 셸 사용 중 (badges + 톱니바퀴)
- 추가 통일 작업 거의 없음
- **이미 만족 상태**

### D. Lab 출력 포맷 통일 (소규모)
- KR "전략별 상세" vs US "전략 상세 모달" 차이
- 둘 다 시장 특화라 통일 의미 작음
- **현재 상태 유지**
