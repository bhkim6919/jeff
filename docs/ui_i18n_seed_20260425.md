## 규약

### 1. 속성 네임스페이스

```
data-i18n="{area}.{kind}.{key}"
```

- `{area}`: 화면/기능 — `dash`, `lab`, `debug`, `common`
- `{kind}`: 역할 — `title`, `label`, `btn`, `badge`, `empty`, `help`
- `{key}`: 의미 식별자 — `holdings`, `profit`, `refresh` 등

### 2. 리소스 파일 위치

```
kr/web/static/i18n/
├── ko.json    # 한국어 (기본)
└── en.json    # 영어
```

JSON 구조는 중첩 객체 (점 표기 매핑):
```json
{
  "dash": {
    "title": {
      "holdings": "보유종목"
    }
  }
}
```

### 3. 태그 규칙

- **단순 텍스트**: 요소 자체에 속성
  ```html
  <h3 data-i18n="dash.title.holdings">보유종목</h3>
  ```
- **복합 요소** (title + badge 등): 텍스트 노드에만 span으로 감싸고 태그
  ```html
  <h3>
      <span data-i18n="dash.title.advisor">AI Advisor</span>
      <span id="advisor-status" class="badge">--</span>
  </h3>
  ```
- **Attribute 값** (title/placeholder): Phase 5에서 `data-i18n-attr` 패턴 추가 예정
- **우선순위**: 속성 존재 → 리소스 lookup. lookup 실패 시 기존 innerText 유지 (폴백).

## Phase 5 실행 시 추가 작업

### 5.1 JS 로더 구현 (`kr/web/static/i18n.js`)

```js
let currentLocale = localStorage.getItem('qtron_lang') || 'ko';
let translations = {};

async function loadLocale(loc) {
    const r = await fetch(`/static/i18n/${loc}.json`);
    translations = await r.json();
    currentLocale = loc;
    localStorage.setItem('qtron_lang', loc);
    applyTranslations();
}

function applyTranslations() {
    document.querySelectorAll('[data-i18n]').forEach(el => {
        const key = el.dataset.i18n;
        const val = resolveKey(key);
        if (val) el.textContent = val;
    });
}

function resolveKey(key) {
    return key.split('.').reduce((o, k) => o?.[k], translations);
}
```

### 5.2 nav 언어 토글 아이콘 추가
현재 KR/US market 토글 + debug 톱니바퀴 옆에 globe 아이콘 추가.

### 5.3 전체 텍스트 태깅
Dashboard, Lab, Unified, Debug 의 모든 **사용자 표시 텍스트** 태깅.
예상: 100~150개 키. 별도 세션 작업.

### 5.4 포맷 로케일
날짜/숫자 포맷은 Intl API 사용 (`Intl.NumberFormat(currentLocale)`).

## 주의

- **지금은 시드만**: `data-i18n` 속성만 추가됐고 로더가 없어 **눈에 보이는 변화 없음**. 브라우저 확인해도 기존 한글 그대로.
- **키 네이밍 변경 금지**: Phase 5 작업 시 이미 태그된 4개 키는 그대로 유지. 변경 시 리소스 파일도 연쇄 수정 필요.
- **US 페이지**: IA 결정 "Phase1은 EN 기준 고정"에 따라 `data-i18n`
  태깅 대상 아님. Phase 5에서 KR 페이지와 리소스 공유 결정 시 확장.
